Dynamic Compression: Self-Pruning Neural Network (CIFAR-10)This repository demonstrates a Dynamic Pruning approach to neural network optimization. Unlike traditional static pruning, this model utilizes learnable gating mechanisms to autonomously identify and eliminate redundant parameters during the training phase.🚀 Project PhilosophyDeep Learning models are notoriously over-parameterized. This project implements a "survival of the fittest" connection strategy, where every weight must justify its existence through contribution to classification accuracy. Connections that fail to provide significant signal are systematically suppressed by an L1-driven gating penalty.🛠️ Technical ArchitectureFour-Layer MLP: A deepened architecture (768-384-128 units) designed to provide a rich parameter space for pruning experiments.Leaky ReLU Activation: Ensures stable gradient flow during aggressive pruning phases.Custom Pruning Gates: * Every weight $W$ is transformed into $W'$ where $W' = W \cdot \sigma(g)$.The gate $g$ is a learnable parameter initialized to start in a semi-active state.Hyperparameter Sensitivity: The model explores the "Sparsity Cliff" using varying $\lambda$ values to find the limit of model compression.📈 Performance & Compression ResultsThe following table summarizes the trade-off between model intelligence and extreme compression:Lambda (λ)Test AccuracySparsity Level (%)Compression Ratio0.0136.16%99.96%~2500:10.0532.73%99.98%~5000:10.1031.25%99.99%~10000:1Analysis: Even at 99.99% sparsity, the model maintains an accuracy 3x higher than random guessing (10%), proving that the pruning mechanism successfully preserved the "core" feature extractors of the network.🧠 The Mathematics of SparsityThe pruning behavior is governed by an L1 Regularization penalty on the sigmoid gates. This creates a constant downward pressure on connection "volume."The "Rent" Analogy: Each connection pays a penalty (rent) to stay active.Automatic Selection: If a connection doesn't reduce classification error enough to offset its rent, the optimizer drives its gate toward zero.Thresholding: We apply a strict threshold of $1 \times 10^{-2}$ to determine the final pruned state.💻 Setup and Usage1.
